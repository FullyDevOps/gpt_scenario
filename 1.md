## 2) Step-by-Step Execution Plan (CONTINUATION ‚Äî fill gaps & add ‚Äúoperational depth‚Äù)

### Stage 7: Lab testing plan + COA workflow (release criteria + frequencies) üß™

**Goal:** every production lot is *held* until results meet *your written specs*.

#### 7.1 Define ‚ÄúLot‚Äù (non-negotiable)

Use **1 lot = 1 SKU + 1 drying run + 1 milling/sieving run + 1 packaging run** (max 1 shift/day recommended).

* Lot ID format: `SKU-YYMMDD-Shift-Run#` (example: `BTR-260208-A-01`).

#### 7.2 Sampling plan (simple, auditable)

For each lot:

1. **Composite sample for lab:** take **10 increments** across the lot (start/middle/end of packaging + multiple cartons), combine into 1 composite.
2. **Retain samples:** keep **2 sealed retains** (see Section 4):

   * Retain A (quality), Retain B (investigation/trace).
3. **Chain-of-custody:** seal bag + sign/date + lot ID + sample weight + sampling points.

#### 7.3 ‚ÄúRelease testing‚Äù panel (per lot)

Minimum panel for **each lot, each SKU**:

* **Moisture %**
* **Water activity (aw)** (critical for low-moisture RTE hygiene approach)
* **Micro:** TAMC, Yeast/Mold, Coliforms, **E. coli**, **Salmonella (absent)**
* **Heavy metals:** Pb, Cd, As, Hg (either per-lot initially or per ‚Äúverification schedule‚Äù below)
* **Foreign matter checks:** sieve record + magnet record + metal detector log (in-house)

> Low-moisture RTE foods have specific sanitation expectations (pathogen prevention mindset). Build sanitation zoning + environmental controls accordingly. ([U.S. Food and Drug Administration][1])

#### 7.4 Verification testing schedule (cost control, still defensible)

After **3 consecutive compliant lots per SKU**:

* **Heavy metals:** move to **quarterly per SKU** (unless you sell into strict markets ‚Üí keep per lot).
* **Pesticide multi-residue:**

  * Conventional: **quarterly per SKU** (or per supplier change).
  * Organic: **every lot initially**, then **monthly per SKU** once stable supplier + certifier acceptance.
* **Aflatoxins:** **annual per SKU** (or if any raw material risk flags appear).
* **Nitrates (beetroot):** optional; do **baseline + annual** unless you plan a claim.

#### 7.5 COA workflow (what happens every lot)

1. **Lot created ‚Üí HOLD status** in inventory system.
2. Packaging completes ‚Üí **finished goods quarantined** (separate physical zone).
3. Lab results received ‚Üí QA compares to written specs.
4. If pass ‚Üí **RELEASE**; if fail ‚Üí **REJECT / REWORK / INVESTIGATE** (CAPA).
5. COA is stored **per lot** and linked to shipment packing list and Amazon inbound batches.

#### 7.6 COA must include (exact fields)

* Supplier/manufacturer name & address; facility ID
* SKU + Lot ID; mfg date; best-by date
* Package type (pouch/jar) + net weight
* Test methods + units + LOQ/LOD where relevant
* Results vs spec limits (show pass/fail)
* Lab name, address, accreditation scope (e.g., ISO 17025), analyst approval
* Sample receive date + report date
